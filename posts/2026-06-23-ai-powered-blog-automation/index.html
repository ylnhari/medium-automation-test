Automating Blog Posting on Medium Using AI Agents and OpenClaw

By ylnhari · June 23, 2026 · automation, blogging, medium, ai-agents, openclaw

This blog post was written, committed, deployed as a static page, and imported as a Medium draft — entirely by an AI agent. No text editor was touched. What you're reading right now was created from a single prompt: "write a blog post about how this automation works".

This post covers the full pipeline, the repo structure that makes it possible, and how to clone it for your own blog.

The Repo: ylnhari/blog

Everything lives in a single GitHub repo at github.com/ylnhari/blog. The structure is minimal:

  • templates/post.html — HTML skeleton with placeholder markers. Every new post starts by copying this.
  • templates/meta.json — Metadata template: title, date, tags, status. No database, just JSON.
  • SETUP.md — Instructions for anyone who wants to fork and repurpose the repo.
  • STANDARDS.md — The HTML contract: which tags are allowed, image rules, structure guidelines. Following these ensures every post imports cleanly into Medium.
  • AGENT.md — The step-by-step workflow for any AI agent that wants to write and publish a post. It covers everything from folder creation to final draft import.
  • posts/YYYY-MM-DD-slug/ — One folder per post, each containing index.html and meta.json. An optional images/ subfolder holds assets served via raw GitHub URLs.
  • posts/index.json — The master registry. Newest entry prepended at the top.

GitHub Pages serves the entire blog at ylnhari.github.io/blog — zero infrastructure, no build tools, just static HTML pushed to main.

How the AI Writes Posts

An AI agent like OpenClaw follows the instructions in AGENT.md and the HTML rules in STANDARDS.md, both of which live in the repo itself. The flow is:

  1. Read AGENT.md and STANDARDS.md to understand the workflow and HTML rules.
  2. Write the post as static HTML — no JavaScript, no inline styles, no divs. Every element maps cleanly to Medium's editor.
  3. Create posts/YYYY-MM-DD-slug/index.html and meta.json.
  4. Prepend an entry to posts/index.json.
  5. Commit all files and push to main.
  6. Wait ~90 seconds for GitHub Pages to deploy.
  7. Navigate to medium.com/p/import, paste the post URL, click Import.
  8. Leave it as a draft — ready for human review and publishing.
The agent never touches the Medium editor. Everything is version-controlled as static files in the repo.

Registering the Skill in OpenClaw

The repo already includes a ready-to-use skill file at skills/create-post/SKILL.md. To register it with an AI agentic system like OpenClaw:

  1. Fork or clone the repo to your machine.
  2. Copy skills/create-post/SKILL.md into your OpenClaw workspace's skills folder.
  3. Open the skill file and customize it for your setup:
# Customizations you'll want to make:
#
# 1. Change `C:\Users\...\blog` to your cloned repo path
# 2. Update trigger phrases to match your preferences
# 3. Add pre-steps: topic confirmation via email, calendar check, etc.
# 4. Add post-steps: notify you on Slack/Telegram when draft is ready
# 5. Tweak the output format to match your reporting style

The skill references AGENT.md and STANDARDS.md from the repo, which means the actual publishing workflow is always in sync with the repo. Update those files, and the agent follows the new instructions automatically — no need to edit the skill file again for content or formatting changes.

Customizing the Workflow

The agent reads its publishing instructions from the repo itself. Want different HTML rules? Edit STANDARDS.md. Want a different commit format or deployment flow? Edit AGENT.md. The skill file only needs to point at your repo — everything else flows from the documentation files inside it.

Fork the repo, adjust AGENT.md to match your writing style, customize skills/create-post/SKILL.md for your local paths and notification preferences, and your AI handles the rest.

Why This Works

Three design decisions make this possible:

  • Static HTML + GitHub Pages = zero ops. No CMS, no database, no hosting fees. Push and it's live.
  • Medium import from URL = no API keys. Medium's import tool ingests any public URL and converts HTML to its editor format. No Medium API tokens needed.
  • Agent reads instructions from the repo = self-documenting. The workflow lives in version control alongside the content. Forking the repo forks the entire publishing pipeline.

The result: a single prompt produces a Medium draft in minutes. No manual steps between idea and draft.
