Why Automate Blogging?

Writing technical content takes time. From research to formatting to publishing, the friction is real. Here's what automation helps with:

  1. Consistency — Keep a regular publishing schedule without manual effort
  2. Formatting — Code blocks, images, and headings handled automatically
  3. Distribution — Cross-posting and social sharing in one click

The Pipeline

The workflow we built has three steps:

  1. An AI agent (DeepSeek, Claude, or any LLM) generates the blog post as a static HTML file
  2. The HTML is committed to this GitHub repo — GitHub Pages auto-deploys it within 90 seconds
  3. The GitHub Pages URL is pasted into medium.com/p/import — Medium imports the full post with formatting preserved

A Quick Code Example

Here's a simple Python function that demonstrates the kind of utility we might publish:

def fibonacci(n: int) -> list[int]:
    """Generate Fibonacci sequence up to n terms."""
    if n <= 0:
        return []
    if n == 1:
        return [0]

    seq = [0, 1]
    for _ in range(2, n):
        seq.append(seq[-1] + seq[-2])
    return seq

# Example usage
result = fibonacci(10)
print(f"First 10 Fibonacci numbers: {result}")

What Medium's Importer Supports

When you import a static HTML page, Medium preserves:

  • All headings (H1, H2, H3)
  • Bold, italic, and inline code
  • Ordered and unordered lists
  • Blockquotes and pull quotes
  • Code blocks (pre/code)
  • Images from absolute URLs — Medium downloads and re-hosts them

What it doesn't support: tables, iframes, JavaScript, and inline CSS. Plan around those from the start.
