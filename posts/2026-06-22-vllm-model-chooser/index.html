vLLM Model Chooser: Find Your Perfect LLM for Any GPU Setup

The Problem: Too Many Models, Too Little Clarity

If you've ever tried to run an LLM on your own GPU hardware, you've hit this wall: a model's parameter count tells you nothing about whether it'll actually fit on your GPU. A 70B model in BF16 needs ~140 GB of VRAM — but in FP8 it needs ~70 GB, and in NVFP4 it squeezes into ~35 GB. That's four wildly different answers for the same model, depending entirely on which quantization format your GPU supports.

The deeper problem? NVFP4 is native on Blackwell (B100/B200), completely absent on Ada Lovelace (L4). MXFP8 works on Blackwell via vLLM SW support but doesn't exist on Hopper. There's no single place to check "which models fit MY hardware" — until now.

Introducing vLLM Model Chooser

vLLM Model Chooser is a single-file interactive web tool that tells you exactly which vLLM-compatible models run on your GPU setup — with zero configuration, no installation, and instant results. It ships as a standalone index.html that you can open in any browser or serve locally.

  • 6 GPU types: L4, A100, H100, H200, B100, B200 — with 1/2/4/8x multi-GPU
  • 108 models with detailed specs: params, VRAM, context length, quantization variants, benchmarks
  • VRAM + quantization dual check: filters by BOTH weight VRAM AND quantization format compatibility
  • GPU info modal: full specs, Native HW vs vLLM SW support table, quantization compatibility matrix
  • vLLM Recipe & HuggingFace links per model

The tool is open-source under MIT license: github.com/ylnhari/vllm-model-chooser

Why Quantization Awareness Matters

The landscape of quantization formats has exploded in the last year. vLLM supports BF16, FP8, INT4/AWQ/GPTQ, NVFP4, MXFP4, and MXFP8 — but not every GPU supports every format:

  • BF16: Native on all GPUs from Ada to Blackwell
  • FP8: Native on Hopper and Blackwell; software emulation on Ampere via vLLM
  • INT4/AWQ/GPTQ: Software support on all architectures via vLLM
  • NVFP4: Native on Blackwell only — completely absent on L4, A100, and H100
  • MXFP4: Native on Blackwell, software on Hopper
  • MXFP8: Software on Blackwell only

This means a model that fits on a B200 in NVFP4 might be 4x too large for the same model on an H100. The Chooser handles this automatically — pick your GPU, and it only shows quantization variants that actually work on that hardware.

How It Works

The interface is simple but powerful:

  1. Select your GPU — L4, A100, H100, H200, B100, or B200
  2. Choose GPU count — 1, 2, 4, or 8 GPUs
  3. Optional filters — Precision format, model type (Text, Vision, MoE, Embedding), minimum context length
  4. Sort — by parameters, VRAM, context length, MMLU score, or HumanEval score
  5. Search — type any model name to find it instantly

    6. The tool instantly recalculates and shows you exactly which models fit, with per-GPU feasibility indicators (green checkmark for fits, red X for doesn't) for each count configuration.

    For each model, you see:

    • Parameter count & active parameters (for MoE models like DeepSeek-V4 with 1600B/49B)
    • VRAM requirements in GB
    • Context length
    • Available quantization variants with VRAM at each quantization level
    • MMLU & HumanEval benchmark scores
    • Direct links to vLLM Recipe and HuggingFace

    Multi-GPU Support: From 1 to 8 GPUs

    One of the most valuable features is the multi-GPU slider. The tool combines usable VRAM across GPUs (with a 95% usable factor to account for system overhead) and checks model fit at each count. This makes it easy to answer questions like:

    • "Can Llama-3.3-70B run on a single L4?" — No (needs 140 GB BF16, L4 has 23 GB usable).
    • "What about 4x L4?" — Yes! 184 GB usable is enough for BF16.
    • "What about 2x H100 in FP8?" — Yes, 152 GB usable comfortably fits 70 GB FP8.

    The tool even handles next-gen hardware: B100 and B200 with 192 GB each (182 GB usable per card) unlock 8-GPU setups with up to 1,456 GB of usable VRAM — enough for models like DeepSeek-V4 (960 GB in NVFP4) or Qwen3-Coder-480B.

    Built for Practical Deployment

    Under the hood, vLLM Model Chooser is refreshingly simple: a single index.html that bundles all HTML, CSS, and JavaScript. No build step, no dependencies, no npm install. Just serve it with any HTTP server:

    python3 -m http.server 5000
# Then open http://localhost:5000

    The model data is sourced from vLLM's official recipe catalog (recipes.vllm.ai/models.json) and supplemented with detailed quantization compatibility information from vLLM's quantization docs.

    If you're deploying LLMs with vLLM, the vLLM Model Chooser saves you hours of VRAM arithmetic and compatibility research. Pick your GPU, see what fits, and get deploying — all from a single HTML file.
